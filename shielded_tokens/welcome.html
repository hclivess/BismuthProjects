Welcome to the Bismuth shielded token interface

Warning! This is an alpha version. Only use for testing.

Glossary:

Shielded token:

A token operating on the Bismuth network with access limited by the master key.

Master key:

A file which contains decryption details for individual Bismuth shielded token transactions.

Description:

Shielded token is a special type of token, which requires you to have the master key for its decryption. Without a master key, you are unable to see transactions on the network. With the master key, you have full access to the transaction history and balances for the given token.

What is yet to be added:

Public transactions:

Also referred to as "ambiguous transactions". These are zero-knowledge transactions that can be performed by people who do not have access to the shielded token master key. Such third parties can blindly submit such transactions in good faith that they are transacting value. Only those with keys can validate such transactions.

Chain-switching protection:

Rigorous protection against double spending and forking has not been implemented yet.

Anchors:

The system is missing anchor points that limit calculation only to the unprocessed part of the chain.
